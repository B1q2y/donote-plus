# 📦 DoNote+ プルリクエスト運用ルール

## ✅ 基本ルール

- 機能単位でブランチを切って作業（例：`feature/task-edit`）
- 作業が完了したら GitHub 上で PR（Pull Request）を作成
- レビュー後に `main` ブランチにマージ（Vercel に自動デプロイ）

## 📋 PRタイトルの書き方

- 絵文字 + 要約（コミットメッセージと同じ形式）

### 例：
✨ タスク完了チェック機能の追加
🐛 削除ボタンが効かないバグを修正
📝 CONTRIBUTING.md を追加

## 💬 PR本文のテンプレ（任意）

### 概要
このPRでは、◯◯の機能を実装しました。

### 詳細
- ◯◯コンポーネントを新規作成
- Supabaseに◯◯カラムを追加
- UIを改善しユーザー操作性を向上

### 動作確認
- ローカルでの動作確認済み
- デプロイ環境でも確認済み（URL付きなら追記）

## 🔁 その他

- 作業中PRにはタイトルに「🚧」をつけておく
- WIPのまま `main` にマージしない
